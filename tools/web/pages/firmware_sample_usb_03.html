usb_03 — string descriptors and per-unit identity

Design + code flow for the third sample in the USB Book series. Keeps usb_02's descriptor tree (device → cfg → iface → 2 bulk EPs) byte-identical, but turns on every string descriptor and replaces the auto-generated chip-ID hex serial with a runtime-formatted EDV-XXXX-YYYY-ZZZZ per-unit identity. The teaching point: descriptor indices are late-bound — the iface descriptor declares iInterface = N, but the actual UTF-16LE bytes for index N are registered separately, at runtime, before usbd_init.

Authoring status — sample built, doc revised post-bring-up

First written 2026-05-08 as a doc-first spec; sample scaffolded and flashed 2026-05-11. Bring-up surfaced one gotcha (G15, see §5) that this revision incorporates, along with the resulting index-assignment shift (iInterface = 4, iConfiguration = 5). The code listings in §8–§9 are the on-disk source at firmware/samples/usb/usb_03/src/; any drift is a bug to fix in both files.

1. Goal & scope

One sentence: turn on every standard string descriptor (LANGID, Manufacturer, Product, Serial, Configuration, Interface) so lsusb -v on the host renders human-readable text at every level — and replace the auto-generated chip-ID hex serial with a runtime-formatted EDV-XXXX-YYYY-ZZZZ, demonstrating that string descriptors are data registered at boot, not code baked in.

Pairs with USB Book Ch 07 — String descriptors & per-unit identity (chapter not yet written; this design doc is the source-of-truth until it is).

The two halves of the chapter

String descriptors is the wire-protocol half: language IDs, UTF-16LE encoding, the iManufacturer / iProduct / iSerialNumber / iConfiguration / iInterface index slots, and the back-and-forth control transfers GET_DESCRIPTOR(STRING, idx, langid). Per-unit identity is the firmware half: how to make every assembled board advertise a serial that is unique to its silicon, with zero per-unit production-line work. usb_03 is the smallest sample that teaches both.

What usb_03 does not cover

  • Persisting the serial to flash / LittleFS — the chip ID is silicon-deterministic, so re-deriving on every boot gives the same value. Persistence only matters when you want to mutate the serial post-boot (provisioning, RMA swap), which is a separate concern. LittleFS belongs to a future "firmware infrastructure" sample where persistence is the topic, not a side dish.
  • Multiple language IDs — we ship LANGID 0x0409 (English-US) only. Adding French, Japanese, etc., is a one-line change to the LANGID array; the more interesting work (selecting the right string at GET_DESCRIPTOR time, providing alternate bytes) is uncommon enough to defer.
  • Bulk/interrupt data flow — still declared-but-not-implemented, same as usb_02. Host bulk reads on EP 0x81 still NAK forever. usb_04 (Ch 08) implements EP0 control transfers in depth; the bulk endpoints get filled in later.
  • BOS / MS OS 2.0 platform-capability descriptors — those live in usb_25 (Ch 23 — modern driverless).

3. The descriptor tree, byte-level (now with strings)

The configuration descriptor tree is byte-identical to usb_02 except for two index references: the configuration descriptor's iConfiguration field is no longer 0, and the interface descriptor's iInterface is no longer 0. Those bytes point at separately-registered string descriptors.

3b. The string descriptors

Six discrete USB descriptors, each fetched by the host with its own GET_DESCRIPTOR(STRING, index, langid) control request. Indices are assigned in registration order by the stack — the 1st usbd_add_descriptor() call gets index 0, the 2nd gets index 1, and so on. Index 0 is reserved for LANGID by spec (USB 2.0 §9.6.7), so we add LANGID first.

One subtlety: only five of the six strings are added by us. cfg_str is added implicitly by usbd_add_configuration() because we passed &cfg_str to USBD_CONFIGURATION_DEFINE — see G15. So in registration order: LANGID, Manufacturer, Product, Serial, Interface, then Configuration last.

Indices are runtime-assigned, not hard-coded

The 0x04 and 0x05 shown above come from the exact registration sequence in main.c (Interface explicit, then Configuration auto-added by usbd_add_configuration()). Add a string in a different order — or call usbd_add_descriptor(&cfg_str) yourself before usbd_add_configuration — and the indices shift. Never hard-code an index in the iface or config descriptor — call usbd_str_desc_get_idx() after registration and patch the field. This is the late-binding moment §4 dwells on.

4. The late-binding moment

This is the conceptual heart of Ch 07. Pause here.

In the byte-level descriptor blob, the interface descriptor advertises iInterface = 4. When the host wants the interface's name, it issues GET_DESCRIPTOR(STRING, 4, langid). The stack walks its registered string descriptors, finds the one at index 4, returns its UTF-16LE bytes. What's registered at index 4 is decided by the firmware, at runtime, before usbd_init().

Three consequences for how we write code:

  1. The descriptor blob in strings_demo.c cannot just write .iInterface = 4 as a literal. If you reorder the usbd_add_descriptor() calls in main.c, the index shifts. You'd need to keep two files in sync by hand. The defense: initialize .iInterface = 0 in strings_demo.c, then patch it from main.c after registering the interface-name string and querying its assigned index.
  2. The serial-number string buffer must exist in writable memory. The Zephyr USBD_DESC_SERIAL_NUMBER_DEFINE macro hands you an immutable struct that auto-formats from hwinfo at request time. We bypass that — we build a custom EDV-XXXX-YYYY-ZZZZ string at boot and write it into a buffer the descriptor node points at. The descriptor node is registered once, but the bytes it points at change between main() entry and the first host enumeration request.
  3. String registration must complete before usbd_init(). The init step walks the descriptor tree and validates every string-index reference. A late-bound iInterface patched after usbd_init would already have validated against the wrong (stale) index. Order is: add all strings → patch index references → usbd_init → usbd_enable.
Mental model

A descriptor is a static blob with index references. A string descriptor is a separate registration flow: each string gets an index slot at registration time. The index references in the descriptor blob are filled in by firmware after the strings are registered, before the stack validates the tree. This separation is what makes per-unit serials, localised product names, and dynamic interface naming all possible without rebuilding firmware.

New for usb_03 (string-specific)

#GotchaDefense
G11LANGID at index 0 is mandatory. Without it the host cannot resolve any other string — every GET_DESCRIPTOR(STRING, n > 0, langid) request specifies a langid the device must claim to support, and the only way to advertise supported langids is the index-0 LANGID descriptor itself. Skip it and iManufacturer = 1 is dead bytes the host will never resolve.USBD_DESC_LANG_DEFINE(lang_desc) defined at file scope; first usbd_add_descriptor() call in main(), before any other string. The Zephyr default is English-US (0x0409), which every host accepts.
G12Wire bytes are UTF-16LE, source code is ASCII7. The USBD_DESC_*_DEFINE macros widen each ASCII byte to a 2-byte UTF-16LE code unit at request time (driven by .ascii7 = true on the usbd_desc_node). Non-ASCII characters in the literal — em dashes, accented letters, emoji — produce wrong-on-the-wire bytes silently. Use plain ASCII or pre-encode UTF-16LE bytes manually with .ascii7 = false.Every string literal in main.c is checked to be ASCII7 (0x20–0x7E). The product string deliberately uses a hyphen -, not an en-dash —. The EDV-XXXX-YYYY-ZZZZ serial uses %02X formatting, which only emits 0–9 and A–F.
G13Index-reference fields must be patched after string registration, before usbd_init. Hard-coding .iInterface = 4 in the descriptor struct at file scope assumes a specific registration order — fragile. Setting it before usbd_init from the result of usbd_str_desc_get_idx() is robust. (iConfiguration is the easier sibling: the stack patches it for us inside usbd_add_configuration() — see G15.)main.c calls strings_demo_set_iface_idx(usbd_str_desc_get_idx(&iface_str)) immediately after adding iface_str and immediately before usbd_add_configuration(). The setter is a one-line export from strings_demo.c.
G14Custom-formatted serial requires bypassing USBD_DESC_SERIAL_NUMBER_DEFINE. That macro hard-codes .use_hwinfo = true, which makes the stack call hwinfo_get_device_id() and emit raw uppercase-hex of every returned byte at request time. There's no hook to format the bytes. Hand-roll a struct usbd_desc_node with .utype = USBD_DUT_STRING_SERIAL_NUMBER and .ascii7 = true, point it at a writable buffer, populate the buffer at boot.See main.c §7 — serial_buf[] is pre-initialised to a placeholder ("EDV-0000-0000-0000") so USB_STRING_DESCRIPTOR_LENGTH evaluates correctly at compile time, then overwritten at boot with the chip-ID-derived value before usbd_add_descriptor(&serial_node).
G15Don't double-add the configuration-string descriptor. USBD_CONFIGURATION_DEFINE(cfg, ..., &cfg_str) stores the pointer in cfg.str_desc_nd; later, usbd_add_configuration() calls usbd_add_descriptor(uds_ctx, cfg.str_desc_nd) on your behalf and writes iConfiguration into the cfg descriptor. If you also call usbd_add_descriptor(&cfg_str) in main(), the node ends up on the descriptors dlist twice — the second add hits the sys_dnode_is_linked() guard at usbd_desc.c:146 and returns -EALREADY (= -120 in Zephyr's minimal libc, where EALREADY = 120; not the Linux glibc value of 114). Surfaced at boot as <err> usbd_cfg: Failed to add configuration string descriptor + <err> usb_03: add cfg: -120. First hit during bring-up 2026-05-11.Register only the strings the stack can't add for you — LANGID, Manufacturer, Product, Serial, Interface. Leave cfg_str to usbd_add_configuration(). With our five explicit adds before it, cfg_str lands at index 5 and iConfiguration is auto-patched to 5.

8. Code: src/main.c

This is where most of the new work lives. Listing is annotated heavily — every block ties back to a numbered step in §7 and a gotcha defense in §5.

firmware/samples/usb/usb_03/src/main.c
/*
 * usb_03 — string descriptors and per-unit identity.
 *
 * Builds on usb_02's descriptor tree with a complete set of standard string
 * descriptors (LANGID, Manufacturer, Product, Serial, Configuration, Interface)
 * and a runtime-formatted EDV-XXXX-YYYY-ZZZZ serial derived from the RP2350
 * 64-bit chip ID. Demonstrates that descriptor-index references are late-bound:
 * the iface descriptor's iInterface field is patched after string registration,
 * before usbd_init() validates the tree.
 *
 * See firmware_sample_usb_03.html for the design narrative + byte-level layout.
 */

#include <stdio.h>                      /* snprintf */
#include <errno.h>
#include <zephyr/kernel.h>
#include <zephyr/usb/usbd.h>
#include <zephyr/drivers/hwinfo.h>       /* hwinfo_get_device_id (G14) */
#include <zephyr/logging/log.h>

LOG_MODULE_REGISTER(usb_03, LOG_LEVEL_INF);

/* G3 defense: single source of truth for the class-registration string. */
#define DEMO_CLASS_NAME "strings_demo"

/* Exported by strings_demo.c — patches iface.iInterface (G13). */
extern void strings_demo_set_iface_idx(uint8_t idx);

/* ===========================================================================
 * Static string descriptors (compile-time literals).
 * Order of registration in main() determines index assignment.
 * =========================================================================*/

USBD_DESC_LANG_DEFINE(lang_desc);                                          /* G11 */
USBD_DESC_MANUFACTURER_DEFINE(mfr_desc,     "edevkit");
USBD_DESC_PRODUCT_DEFINE     (product_desc, "usb_03 - strings_demo");      /* G12: ASCII7 only */
USBD_DESC_CONFIG_DEFINE      (cfg_str,      "Default config");
USBD_DESC_STRING_DEFINE      (iface_str,    "Vendor (descriptor demo, no I/O)",
                              USBD_DUT_STRING_INTERFACE);

/* ===========================================================================
 * Custom serial-number string descriptor (G14).
 *
 * Hand-rolled struct usbd_desc_node + writable buffer. The descriptor's
 * .ptr is initialised once at file scope to point at serial_buf; the buffer
 * itself is overwritten at boot with the chip-ID-derived value.
 *
 * USB_STRING_DESCRIPTOR_LENGTH(s) = sizeof(s) * 2  (verified in usbd.h:56).
 * For "EDV-XXXX-YYYY-ZZZZ" sizeof = 19, bLength = 38, layout: 2-byte header
 * + 18 UTF-16LE code units.
 * =========================================================================*/

#define SERIAL_TEMPLATE   "EDV-0000-0000-0000"        /* placeholder, same length as runtime value */
#define SERIAL_BUF_SIZE   sizeof(SERIAL_TEMPLATE)     /* 19 incl. NUL */

static char serial_buf[SERIAL_BUF_SIZE] = SERIAL_TEMPLATE;

static struct usbd_desc_node serial_node = {
    .str = {
        .utype  = USBD_DUT_STRING_SERIAL_NUMBER,
        .ascii7 = true,                               /* G12 */
    },
    .ptr             = serial_buf,
    .bLength         = USB_STRING_DESCRIPTOR_LENGTH(SERIAL_TEMPLATE),
    .bDescriptorType = USB_DESC_STRING,
};

/* Read the RP2350 64-bit chip ID and format the first 6 bytes (12 hex chars)
 * into the serial_buf. Same sizeof, same bLength — only the bytes change. */
static int serial_init(void)
{
    uint8_t id[8] = {0};
    ssize_t n = hwinfo_get_device_id(id, sizeof(id));

    if (n < 6) {
        LOG_ERR("hwinfo_get_device_id returned %d (need >= 6)", (int)n);
        /* Don't fail boot — the placeholder "EDV-0000-0000-0000" is still a
         * valid string. Useful in CI / emulator where HWINFO might be stub. */
        return -EIO;
    }

    int written = snprintf(serial_buf, sizeof(serial_buf),
                           "EDV-%02X%02X-%02X%02X-%02X%02X",
                           id[0], id[1], id[2], id[3], id[4], id[5]);
    if (written != (int)(sizeof(serial_buf) - 1)) {
        LOG_ERR("serial format truncated (wrote %d, expected %d)",
                written, (int)(sizeof(serial_buf) - 1));
        return -ENOSPC;
    }

    LOG_INF("serial: %s", serial_buf);
    return 0;
}

/* ===========================================================================
 * Configuration descriptor.
 *
 * Third arg is now &cfg_str (was NULL in usb_02). It is stored as
 * cfg.str_desc_nd; usbd_add_configuration() will register cfg_str for us and
 * write back iConfiguration with the resulting index.
 *
 * G15 — do NOT also call usbd_add_descriptor(&cfg_str) yourself. The stack
 * does it inside usbd_add_configuration(); a manual pre-add lands cfg_str on
 * the descriptors dlist, and the stack's second add returns -EALREADY (-120
 * in Zephyr's minimal libc) — surfaced as "Failed to add configuration
 * string descriptor" + "add cfg: -120".
 * =========================================================================*/

USBD_CONFIGURATION_DEFINE(cfg, USB_SCD_REMOTE_WAKEUP, 50, &cfg_str);

/* ===========================================================================
 * Device-level identity. PID bumped to 0xede3; bcdDevice bumped to 0x0103
 * to invalidate the macOS descriptor cache after coming from usb_02.
 * =========================================================================*/

USBD_DEVICE_DEFINE(usbd_ctx, DEVICE_DT_GET(DT_NODELABEL(usbd)),
                   0x1209, 0xede3);

int main(void)
{
    int err;

    /* (0) Generate the per-unit serial BEFORE registering the descriptor. */
    (void)serial_init();        /* placeholder is a valid fallback if this errors */

    /* (1)–(5) Register string descriptors that have no auto-add path.
     *         Order assigns indices. G7 — order matters; G11 — LANGID first.
     *         cfg_str is intentionally NOT added here: usbd_add_configuration()
     *         registers it on our behalf (see G15 next to USBD_CONFIGURATION_DEFINE). */
    err = usbd_add_descriptor(&usbd_ctx, &lang_desc);
    if (err) { LOG_ERR("add lang: %d", err);          return -1; }
    err = usbd_add_descriptor(&usbd_ctx, &mfr_desc);
    if (err) { LOG_ERR("add mfr: %d", err);           return -1; }
    err = usbd_add_descriptor(&usbd_ctx, &product_desc);
    if (err) { LOG_ERR("add product: %d", err);       return -1; }
    err = usbd_add_descriptor(&usbd_ctx, &serial_node);
    if (err) { LOG_ERR("add serial: %d", err);        return -1; }
    err = usbd_add_descriptor(&usbd_ctx, &iface_str);
    if (err) { LOG_ERR("add iface_str: %d", err);     return -1; }

    /* (5a) G13 — patch the iface descriptor's iInterface field to whatever
     *      index the stack just assigned to iface_str. MUST happen before
     *      usbd_init() validates the tree.                                   */
    uint8_t iface_idx = usbd_str_desc_get_idx(&iface_str);
    LOG_INF("iface_str registered at idx %u", iface_idx);
    strings_demo_set_iface_idx(iface_idx);

    /* (6) Add the configuration. The stack auto-registers cfg_str (next idx)
     *     and writes back iConfiguration in cfg_desc — we never touch either. */
    err = usbd_add_configuration(&usbd_ctx, USBD_SPEED_FS, &cfg);
    if (err) { LOG_ERR("add cfg: %d", err); return -1; }

    /* (7) Attach the strings_demo class instance to configuration 1. */
    err = usbd_register_class(&usbd_ctx, DEMO_CLASS_NAME,
                              USBD_SPEED_FS, 1);
    if (err) { LOG_ERR("register class: %d", err); return -1; }

    /* (8) Validate + finalize. The stack walks every class's fs_desc[] AND
     *     re-validates every iInterface / iConfiguration index reference now. */
    err = usbd_init(&usbd_ctx);
    if (err) { LOG_ERR("usbd_init: %d", err); return -1; }

    /* (9) Connect the pull-up. Host can now see us. */
    err = usbd_enable(&usbd_ctx);
    if (err) { LOG_ERR("usbd_enable: %d", err); return -1; }

    LOG_INF("usb_03 strings_demo: enabled, idle on EP0");

    while (true) {
        k_sleep(K_FOREVER);
    }
    return 0;
}

In usb_02, USBD_DESC_SERIAL_NUMBER_DEFINE's BUILD_ASSERT(IS_ENABLED(CONFIG_HWINFO), ...) would have failed the build if HWINFO weren't already enabled by some upstream default. usb_03 doesn't use that macro — it calls hwinfo_get_device_id() directly — so the dependency is now ours to declare. 13. Anticipated pitfalls

Symptom → Fix

"Boot logs show <err> usbd_cfg: Failed to add configuration string descriptor followed by add cfg: -120, and the device never enumerates." G15: you're double-adding cfg_str. Either remove the explicit usbd_add_descriptor(&usbd_ctx, &cfg_str) in main() (the stack adds it inside usbd_add_configuration() because you passed &cfg_str to USBD_CONFIGURATION_DEFINE), or pass NULL as the fourth macro arg if you really want to manage the registration yourself. Note: -120 is -EALREADY in Zephyr's minimal libc, not -ENOMSG — Zephyr's errno.h values diverge from Linux glibc.

Symptom → Fix

"Manufacturer / Product / Serial all show up as empty strings or as (error) in lsusb." The LANGID descriptor at index 0 is missing, malformed, or registered AFTER another string (G11). The host requested GET_DESCRIPTOR(STRING, 0, 0) first, got nothing useful, and now refuses to issue any further string requests. Confirm USBD_DESC_LANG_DEFINE(lang_desc) exists and is the first usbd_add_descriptor() call in main().

Symptom → Fix

"iInterface = 0 on the host even though we registered the interface string." The patch in step (5a) didn't run, or ran after usbd_init() (G13). Check the call order in main.c: usbd_str_desc_get_idx(&iface_str) + strings_demo_set_iface_idx(...) must come before usbd_add_configuration(). Add a LOG_INF("iface_idx = %u", idx) to confirm.

Symptom → Fix

"Serial shows as EDV-0000-0000-0000 on the host." The serial_init() call ran but hwinfo_get_device_id() returned -ENOSYS (no driver) or fewer than 6 bytes. RTT will show the corresponding LOG_ERR from serial_init. Verify CONFIG_HWINFO=y in prj.conf and that the active board has a registered hwinfo driver. RP2350's hwinfo driver lives in $ZEPHYR_BASE/drivers/hwinfo/hwinfo_rpi_pico.c; if you've ported usb_03 to a different board, check that target.

Symptom → Fix

"Build error: BUILD_ASSERT failed: HWINFO not enabled." You left USBD_DESC_SERIAL_NUMBER_DEFINE in main.c from a copy-paste of usb_02 in addition to the new hand-rolled serial_node — and forgot the CONFIG_HWINFO=y. Either remove the macro use (we replaced it deliberately for G14) or add CONFIG_HWINFO=y. Don't keep both.

Symptom → Fix

"Product string ends mid-word in lsusb (truncation)." The string literal contains a non-ASCII character (em dash, accented letter) and the .ascii7 = true path widened it to a wrong UTF-16LE code unit (G12). lsusb may render the truncation as a stray glyph or stop printing partway through. Replace non-ASCII characters with ASCII7 substitutes (— → -, é → e) or pre-encode UTF-16LE bytes manually with .ascii7 = false.

Symptom → Fix

"Two boards on the same host show identical serials." Either (a) hwinfo_get_device_id() returned a board-default constant on both (driver bug, very rare on RP2350), or (b) you're running on an emulator where the chip ID is stubbed. On real RP2350B silicon the 64-bit unique ID is laser-fused and globally unique. Compare the chip IDs in RTT (id[0..7]) before formatting — if those differ but the formatted serials match, you have a snprintf formatting bug.

Symptom → Fix

"usbd_init: -22 (-EINVAL) at boot, but everything compiled." Most likely cause specific to usb_03: the iInterface value patched in step (5a) is out of range — for instance, strings_demo_set_iface_idx(0) got called by accident, or usbd_str_desc_get_idx(&iface_str) returned 0 because iface_str wasn't actually added. Add LOG_INF on the returned index and re-flash.

Symptom → Fix

"After flashing usb_03, lsusb -v still shows usb_02's strings." Host descriptor cache. Hosts cache descriptors keyed by {VID, PID, bcdDevice}; if any one of those didn't change, the OS will not re-read. usb_03 bumps bcdDevice to 0x0103 precisely to invalidate the cache after coming from usb_02's 0x0100. If you're iterating within usb_03 and changed a string without bumping bcdDevice, increment bcdDevice by one (e.g., 0x0103 → 0x0104) for that build.
