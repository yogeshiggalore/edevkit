Glossary

USB terminology, distilled.

Status

This appendix is filled out as terms appear in chapters. Each chapter that introduces a new term adds a definition here. Once Part I ships, expect ~30 entries; once the book is complete, ~120.

Until then, a few high-frequency terms readers tend to look up first:

A few starters

BOS (Binary Object Store) — A USB 3.x-introduced descriptor format that lets a device declare extended capabilities (including driverless-Windows hooks via Microsoft OS 2.0 platform descriptor). Required for modern WinUSB binding. See Chapter 09.

IAD (Interface Association Descriptor) — Glue descriptor that tells the host "interfaces N..N+k belong together as one logical function." Without it, composite devices like dual-CDC fall apart on Windows. See Chapter 18.

MPS (Max Packet Size) — The largest single-transaction payload the endpoint accepts. 8/16/32/64 for FS endpoints, 512 for HS bulk. Wrong MPS = silent data corruption. See Chapter 13.

NRZI (Non-Return-to-Zero Inverted) — The line-coding USB uses on D+/D-. A '0' bit toggles the line; a '1' bit holds it. Combined with bit-stuffing, this provides clock recovery. See Chapter 02.

SETUP packet — The 8-byte fixed-size token that opens every control transfer. Carries bmRequestType, bRequest, wValue, wIndex, wLength. See Chapters 03 and 08.

ZLP (Zero-Length Packet) — A bulk transfer that's an exact multiple of MPS needs an empty terminating packet so the host knows the transfer is complete. Forgetting it is the #1 silent-stall bug. See Chapter 13.
